Скрипты для работы с сайтом parser.im

app1.py - исправленный файл скрипта подписки. На входе забирает файл, считает частотность хештегов, убирает частность ниже определенной границы. Входной файл - hashtags.txt. Помимо файла result.txt с частотностью он создает файл tag.txt в котором одни хештеги (без цифр), необходимый для скрипта app_while.txt

app_while.py - доработанный скрипт, который в автоматическом режиме отправляет на проверку частотности хэштеги из файла tag.txt. Когда скрипт получает информацию о том, что задание выполнено, то результат сохраняется в файл 	в режиме дОзаписи.

После прохода 1 цикла файл ~taskid.txt  перезаписывается.

Переменные:

part_list = количество циклов, которые отработает программа. Желательно выставлять кратное 100. Например, если у вас в файле tag.txt 400 хештегов, то количество циклов делаем 4.
Также в самом файле tag.txt количество хештегов желательно делать кратное 100. Например, 200, 300, 600 и так далее.
Параметр sleep(420) - в скобках время в секундах, которое ждет программа, прежде чем снова проверить статус заданий. Как правило, одно задание выполняется в районе 5-6 минут.  
 
	
apikey.py - модуль с основными параметрами, такими как url, названия файлов и т.д.


Желательно создавать новый каталог для каждой партии заданий и копировать файлы скрипта туда.
- При добавлении задания обязательно наличие файла tag.txt
- При проверке статусов - ~taskid.txt
